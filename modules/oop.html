1. S3: The Informal Standard

The Concept: S3 is like a Stars Hollow town meeting: informal, flexible, and relies on everyone just "knowing" what to do. It uses generic functions that decide what to do based on the class of the object.

Syntax: class(obj) <- "myclass"

You can turn any list into an object just by giving it a class name. Then, you define methods like print.myclass.

2. S4: The Strict Standard

The Concept: S4 is Taylor Doose. It has rules, forms, and strict validation. You can't just slap a class name on a list; you have to formally define the class and its "slots" (fields) using setClass.

Syntax: setClass("Name", slots = list(field="type"))

It ensures that an object always has the correct data types. No more chaos.

The Concept: R6 is the Dragonfly Inn. Everything‚Äîdata and methods‚Äîis contained within a single object. Unlike S3 and S4, R6 objects are mutable (they can be changed in place).

Syntax: obj$method()
